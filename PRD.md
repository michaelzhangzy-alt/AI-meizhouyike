# PRD（需求定版 / 验收口径）

本文档用于“定版与验收”：明确目标、角色、范围、验收标准、数据口径、权限边界，并把当前仓库所体现的阶段一交付物固化下来，同时给出阶段二需求拆解与优先级。

## 1. 目标（Goals）

### 1.1 业务目标

- 用官网承接流量并完成“报名留资”转化（核心链路：访问 → 报名/领资料 → 留下联系方式）。
- 建立内容沉淀：课程（公开课/直播课/回放）与资讯（原创/外链导读）持续更新。
- 提供运营后台：课程/资讯/师资/分享配置/站点内容/线索导出等日常运营能力。
- 提供 AI 工具入口：用于增长或提升站内停留与转化（现状包含流式 AI 工具）。

### 1.2 非目标（Non-goals）

（可根据实际运营策略调整）

- 不承诺复杂电商闭环（支付、订单、发票等）。
- 不承诺完整会员订阅体系（仅提供“老付费用户识别/群二维码”能力基础）。
- 不承诺站内完整社区/评论体系。

## 2. 用户角色（Personas）

- 访客用户（匿名）：浏览内容、报名留资、使用部分 AI 工具。
- 运营/管理员（authenticated）：登录后台，管理内容与配置，导出线索，维护会员与群二维码。
- （阶段二候选）内容编辑（细分权限）：仅可管理课程/资讯，不可查看线索或成员信息。

## 3. 功能范围（Scope）

### 3.1 前台（Public）

- 入口与导航：Console 主页 + 顶部导航 + 移动端导航。
- 内容浏览：
  - 公开课：本周公开课页（时间窗口筛选 + 报名）。
  - 课程：列表（系列筛选）+ 详情页。
  - 资讯：列表（类型筛选）+ 详情（阅读量、最近文章、上一篇/下一篇、外链导读）。
  - 关于我们：内容由后台可配置。
- 转化：
  - 报名/领取资料弹窗（姓名 + 手机号）提交线索。
  - 线上环境微信分享（按路径配置分享卡片）。
- AI 工具：
  - AI 工具首页 + Markdown 内容详情页。
  - 流式 AI 工具页面（通过 Edge Function）。

### 3.2 后台（Admin CMS）

- 登录与会话：Supabase Auth（邮箱/密码）。
- 仪表盘：核心指标概览（线索、发布内容等）。
- 线索管理：列表查看、导出 Excel。
- 付费会员：维护老付费成员名单；维护微信群二维码（上传/删除/状态）。
- 内容管理：师资、课程、资讯 CRUD（含富文本编辑、图片上传裁剪）。
- 分享配置：按 `page_path` 配置分享标题/描述/图片、启用状态。
- 站点内容：关于我们/会员权益等页面碎片配置。

## 4. 权限边界（Permissions）

### 4.1 产品口径（应满足）

- Public（匿名）：
  - 可读：已发布课程/文章、启用的分享配置、公共师资与页面碎片。
  - 可写：报名线索（仅写入，不应允许匿名读取线索）。
- Admin（authenticated）：
  - 可读写：所有内容与配置；可查看所有线索与成员信息；可上传/删除存储文件。

### 4.2 现状与差距（需要验收时明确）

- 现状迁移中 `leads` 存在匿名可 SELECT 的策略，会导致线索可被匿名读取风险（与上述产品口径冲突）。  
  - 阶段一验收可选择“允许匿名读取（仅用于开发演示）”或“必须修复后才算通过”。  
  - 建议作为阶段二 P0 安全整改项。

## 5. 数据口径（Metrics Definitions）

用于仪表盘/报表/运营复盘，建议统一口径如下：

- 线索（Lead）：
  - 定义：`leads` 表一条记录即 1 条线索。
  - 线索状态：`leads.status`（`new/contacted/converted/lost`）。
  - 线索来源：`leads.source`（由前端传入，如 navbar、weekly_class、course_detail_hero 等）。
  - 线索时间：`leads.created_at`（统一用 UTC 存储，展示按运营时区转换）。
- 已发布课程数：`courses.status='published'` 的数量。
- 已发布文章数：`articles.status='published'` 的数量。
- 文章阅读量：`articles.views`（通过 RPC `increment_article_views` 累加）。
- 分享配置覆盖率（建议指标）：
  - 定义：前台页面中可被 `share_configs` 命中配置的比例。
  - 备注：动态路由目前难以精确配置，建议阶段二优化匹配策略后再纳入指标。

## 6. 验收标准（Acceptance Criteria）

### 6.1 前台验收

- 路由可直达：
  - 任意前台路由（/home、/news、/courses 等）在生产环境可直接刷新访问不 404（SPA rewrite 生效）。
- 内容展示正确：
  - 课程列表仅展示已发布课程；课程详情可打开并展示富文本内容。
  - 资讯列表仅展示已发布文章；资讯详情可打开并展示内容；阅读量会增加。
  - 外链导读文章：点击后可跳转到外部链接，并展示导读字段（若配置）。
  - 关于我们：后台修改后前台即时生效（无需发版）。
- 报名留资可用：
  - 弹窗校验手机号 11 位；提交成功后展示成功态。
  - 提交后数据库能看到新增 `leads` 记录（字段包含 source、page_url、user_agent 等）。
- 微信分享（线上环境）：
  - 非 localhost 环境打开任一页面，会按 `share_configs` 设置微信分享卡片（若页面存在配置）。
  - 未配置页面应有默认降级文案，不阻塞页面访问。
- AI 工具：
  - AI 工具页能打开并完成一次成功生成（以线上可用的 Edge Function 为准）。
  - 流式输出无明显乱码/断流（以可用性为准，不强制追求“逐 token”一致性）。

### 6.2 后台验收

- 登录与鉴权：
  - 未登录访问任一 `/admin/*` 路由会跳转到 `/admin/login`。
  - 登录后可访问仪表盘与各管理页面。
- CMS 功能：
  - 课程/资讯/师资可新增、编辑、发布/下线（以 status 字段驱动前台展示）。
  - 图片上传可用（上传后前台可展示）。
- 线索管理：
  - 列表可查询（按时间倒序），导出 XLSX 可用。
- 分享配置：
  - 可新增/编辑/启用/禁用；前台微信分享会按配置生效（对静态路由）。
- 站点内容：
  - 可编辑关于我们与会员权益内容；前台实时展示。
- 付费会员与群二维码：
  - 可维护 `paid_members`；可上传/删除群二维码记录（`wechat_group_qr`）。

## 7. 阶段一已完成清单（以仓库现状为准）

- 前台：Console 主页、首页、本周公开课、课程列表/详情、资讯列表/详情、关于我们、AI 工具页。
- 转化：报名弹窗提交 leads（RPC 优先 + fallback insert）。
- 内容系统：课程/资讯/师资在后台可 CRUD，并通过 status 控制前台展示。
- 运营配置：微信分享配置（share_configs）、站点内容碎片（page_fragments）。
- 数据层：Supabase migrations（表结构、RLS、Storage buckets、RPC）。
- 部署：Vercel SPA rewrite（保证前端路由直达）。

## 8. 阶段二需求拆解与优先级

说明：阶段二以“安全闭环 + 运营效率 + 可观测性 + 体验提升”为主，按优先级从高到低列出。

### P0（必须做）

- 安全整改：线索隐私
  - 修订 `leads` RLS：匿名仅允许 INSERT，不允许 SELECT；后台 authenticated 才能 SELECT/UPDATE/DELETE。
  - 若需要前台展示“本人提交记录”，需引入用户体系与按用户过滤策略。
- 密钥治理：
  - Edge Function 禁止硬编码第三方密钥；统一迁移到环境变量/Secrets，并做密钥轮换。
- 报名成功页二维码闭环：
  - 若 RPC 返回 `qr_info`，前端应渲染真实二维码与文案；并明确对“新用户/老付费用户”的差异化提示。

### P1（强烈建议）

- 动态路由分享配置策略：
  - 支持 `share_configs` 的“前缀匹配/通配符/规则优先级”，覆盖 `/news/:id`、`/courses/:id`、`/ai-tools/:slug`。
- 课程系列运营能力：
  - 增加后台 `course_series` 管理页面（CRUD、排序、启用/禁用）。
- 可观测性最低闭环：
  - 前端增加 ErrorBoundary + 错误上报；Edge Function 统一错误返回格式与 requestId。
- 数据口径完善：
  - 仪表盘增加线索状态分布、来源分布；并明确时间范围与时区口径。

### P2（体验优化）

- 埋点接入：
  - 接入现有 analytics 封装（PV/事件），并在 index.html 注入 gtag（若选择 GA4）。
- 性能与 SEO：
  - 图片尺寸与加载策略统一；关键页面 SEO（标题/描述/结构化数据）补齐。
- 内容编辑体验：
  - 富文本编辑器图片资源管理（复用、删除、替换）与草稿自动保存。

## 9. 风险与依赖（Risks & Dependencies）

- 微信分享依赖公众号与域名白名单配置；签名接口依赖 WeChat AppId/Secret 正确注入。
- AI 工具依赖第三方模型服务稳定性与限流策略；需要备用方案与降级提示。
- Supabase RLS 修改属于“数据安全变更”，需要上线前完整回归与最小权限校验。
